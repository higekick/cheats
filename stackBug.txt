・gradleの記述
モジュールのプロジェクト
sdk_hogeのgradleに以下のように記述していた

sdk_hoge直下にhoge-debug.aarがある

configurations.maybeCreate("default")
artifacts.add("default", file('dprofiler-debug.aar'))


リリースビルドでhoge-release.aarを参照したことになった。

このプロジェクトのgradleに以下のようにかいてもだめだった

    debugCompile(name:'hoge-debug', ext:'aar')
    releaseCompile(name:'hoge-release', ext:'aar')

appのgradleに以下のように書いたらいけた

repositories {
    mavenCentral()
    flatDir {
        dirs 'libs' //this way we can find the .aar file in libs folder
    }
}

dependencies {
    debugCompile(name:'hoge-debug', ext:'aar')
    releaseCompile(name:'hoge-release', ext:'aar')
}

・aarファイルのビルド
  BuildからRebuild Projectとやっていたらうまくaarができていなかった。
  debug/releaseのそれぞれのaarをちゃんと作るには右のGradleのタブから
  対象プロジェクト->Tasks->build->assembleもしくは、assembleDebug、assembleReleaseとやる。
　対象プロジェクトの\build\outputs\aarの下にそれぞれできている。
